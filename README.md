# Агент приоритизации документов

AI-агент на основе Claude для управления входящим документооборотом.  
Реализует MVP согласно PRD: анализ → зона ответственности → маршрутизация → приоритизация → дашборд.

## Быстрый старт

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Задать API-ключ
set ANTHROPIC_API_KEY=sk-ant-...

# 3. Запустить демо (6 образцовых документов)
python main.py demo

# Демо для другого сотрудника (финансовый отдел)
python main.py demo --employee petrov_dm

# Фильтр только высокоприоритетных
python main.py demo --priority HIGH

# Анализ одного файла
python main.py file путь/к/документу.pdf

# Анализ папки
python main.py folder ./inbox --employee sidorova_ev
```

## Архитектура

```
main.py          — CLI-интерфейс и точка входа
pipeline.py      — оркестратор полного цикла обработки
agent.py         — Claude API: анализ документа (adaptive thinking)
document_parser.py — извлечение текста из PDF/DOCX/EML
zone_matcher.py  — сравнение с профилями зон ответственности
prioritizer.py   — взвешенный расчёт приоритета
dashboard.py     — Rich CLI-дашборд
models.py        — модели данных
config.py        — настройки (модель, веса, пороги)
zones_config.json — профили зон ответственности
demo_documents.py — образцовые документы для демо
```

## Поддерживаемые форматы

- PDF (требует `pdfplumber` или `pypdf`)
- DOCX (требует `python-docx`)
- EML / текстовые письма
- TXT

## Приоритеты

| Уровень | Порог | Действие |
|---|---|---|
| 🔴 Высокий | ≥ 0.65 | Рассмотреть сегодня |
| 🟡 Средний | ≥ 0.35 | В течение рабочей недели |
| 🟢 Низкий | < 0.35 | Плановая очередь |

Веса критериев (настраиваются в `config.py`):
- Срок ответа: 35%
- Тип документа: 30%
- Сумма: 15%
- Статус контрагента: 12%
- Повторное обращение: 8%
